# 💱 CBR Currency Integration

**Интеграция валютных данных ЦБ РФ в систему учёта**  
Модуль для автоматического ежедневного получения курсов валют с официального API Центрального Банка России, сохранения истории в базу данных и визуализации динамики изменений на интерактивном графике.

---

## 📝 Описание проекта

**Задача:**  
Создать модуль, который ежедневно запрашивает актуальные курсы валют с API ЦБ РФ, сохраняет их в базу данных и выводит динамический график изменений для системы учёта.

**Решение:**  
Разработан веб-сервис на FastAPI с планировщиком APScheduler. Каждый день в 10:00 МСК сервис обращается к `http://www.cbr.ru/scripts/XML_daily.asp`, парсит XML, извлекает курсы (USD, EUR, CNY и др.) и сохраняет их в SQLite. Пользователь через веб-интерфейс может выбрать валюту и период (7/30/90/180 дней), увидеть график изменения курса и ключевую статистику (текущий курс, максимум, минимум, процент изменения). Также доступен ручной запуск синхронизации через кнопку или API.

**Результат:**  
Полностью рабочий автономный модуль, готовый к встраиванию в любую систему учёта.

---

## 🏗 Архитектура и компоненты

Проект состоит из следующих модулей:

| Компонент | Файл | Описание |
|-----------|------|-----------|
| **FastAPI приложение** | `main.py` | Точка входа, маршруты, статика |
| **Клиент API ЦБ РФ** | `cbr_client.py` | Запрос к XML, парсинг, преобразование в словарь курсов |
| **Сервис синхронизации** | `currency_service.py` | Логика «получить курсы → сохранить в БД → залогировать» |
| **Работа с БД** | `database.py` | Инициализация таблиц, вставка, выборка, логирование |
| **Планировщик** | `scheduler.py` | APScheduler с cron-триггером на 10:00 МСК |
| **Фронтенд** | `static/index.html`, `style.css`, `script.js` | Адаптивный интерфейс, график Chart.js, статистика |

**Взаимодействие:**  
- Планировщик вызывает `sync_currency_rates()` → запрос к ЦБ → сохранение в БД.  
- Пользователь через браузер обращается к `/` → фронтенд загружает список валют через `/api/currencies` → при выборе периода загружает данные через `/api/graph` и строит график.  
- Ручная синхронизация через POST `/api/sync` вызывает ту же функцию, что и планировщик.
